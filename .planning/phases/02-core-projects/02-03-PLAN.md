<task type="auto">
  <name>Task 1: Create projects router with CRUD operations</name>
  <files>backend/src/routes/projects/projects.ts</files>
  <action>
Create a new file backend/src/routes/projects/projects.ts with Fastify routes:

**POST /api/projects** - Create project:
- Body: { name, leadTeamId, startDate?, endDate? }
- Generate projectId using generateProjectId()
- Auto-create entry in project_teams with isLead=true and effortSize='M' (default)
- Return full project object with version=1

**GET /api/projects** - List all projects:
- Return array of projects with joined status, lead team names
- Include isStopped filter support: ?stopped=true/false
- Order by createdAt desc

**GET /api/projects/:id** - Get single project:
- Return project with:
  - Core fields
  - Status object (name, color)
  - Lead team object
  - All project_teams with team names and sizes
  - All project_values with outcome names
  - All project_change_impact with team names
- Return 404 if not found

**PUT /api/projects/:id** - Update project:
- Body: { ...fields, expectedVersion }
- Use optimistic locking: WHERE id = :id AND version = :expectedVersion
- Increment version on success
- Return 409 with current data if version mismatch
- Return 404 if project not found
- Set updatedAt and updatedBy (from req.user.email or 'dev-user')

**DELETE /api/projects/:id** - Delete project:
- Check for actuals (placeholder: always allow for now, actuals table doesn't exist yet)
- Return 400 with message if actuals exist (future implementation)
- Cascade deletes project_teams, project_values, project_change_impact via FK

**PATCH /api/projects/:id/stop** - Stop project:
- Set isStopped = true
- Increment version
- Return updated project

**PATCH /api/projects/:id/reactivate** - Reactivate project:
- Set isStopped = false
- Increment version
- Return updated project

Use existing patterns from admin routes (db plugin, error handling).

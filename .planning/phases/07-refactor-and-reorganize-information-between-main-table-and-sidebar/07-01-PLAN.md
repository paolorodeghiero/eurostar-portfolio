    **Currency Model Fix:**
    The research clarifies: ALL monetary amounts must be stored in EUR. The budgetCurrency field is deprecated. reportCurrency is display preference only.

    1. In `budget.ts` PUT endpoint:
       - Accept optional `inputCurrency` parameter (defaults to EUR)
       - If inputCurrency is not EUR, convert opexBudget and capexBudget to EUR before saving
       - Always store EUR values in database
       - Log conversion for debugging: "Converting {amount} {inputCurrency} to EUR: {result}"

    2. In `projects.ts` GET / endpoint:
       - Accept query param `reportCurrency` (defaults to EUR)
       - After fetching projects, calculate actualsTotal from receipts table (SUM of amounts WHERE projectId matches, converted to EUR)
       - If reportCurrency !== EUR, convert opexBudget, capexBudget, actualsTotal to reportCurrency before returning
       - Return new fields: `description`, `actualsTotal`, ensure `updatedAt` is included
       - Return `budgetTotal` as computed: opexBudget + capexBudget (in requested currency)

    3. In `currency-converter.ts`:
       - Add helper `convertBatchCurrency(db, amounts: {amount: string, currency: string}[], targetCurrency: string, referenceDate?: string)` for efficient batch conversion
       - Ensure convertCurrency handles null amounts gracefully (return null)

    **API Response Shape:**
    ```typescript
    {
      // existing fields...
      description: string | null,
      budgetTotal: string | null,  // opex + capex in reportCurrency
      actualsTotal: string | null, // sum of receipts in reportCurrency
      updatedAt: string,
    }
    ```

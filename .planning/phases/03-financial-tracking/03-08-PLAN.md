  <how-to-verify>
**Setup:**
1. Start backend: cd backend && npm run dev
2. Start frontend: cd frontend && npm run dev
3. Open browser to http://localhost:5173

**Test 1: Budget Lines Import (BLIN-01, BLIN-02, BLIN-03)**
1. Navigate to Admin > Budget Lines
2. Create a test Excel file with columns: Company, Department, CostCenter, LineValue, Amount, Currency, Type, FiscalYear
3. Add rows with valid data (use existing departments and cost centers)
4. Click "Import Excel" and upload file
5. Verify: Imported count shown, table displays new budget lines
6. Verify: Allocated shows 0, Available shows full amount

**Test 2: Project Budget Setup (BUDG-01, BUDG-05)**
1. Open a project in the sidebar
2. Navigate to Budget tab
3. Set Currency to EUR (or matching your budget lines)
4. Set OPEX Budget: 50000
5. Set CAPEX Budget: 30000
6. Verify: Total shows 80000
7. Verify: Cost T-shirt auto-updates based on thresholds

**Test 3: Budget Allocation (BUDG-02, BUDG-03, BUDG-04)**
1. In Budget tab, click "Add Allocation"
2. Search and select a budget line
3. Enter allocation amount that exceeds available
4. Verify: Error message shows "Exceeds available amount"
5. Enter valid allocation amount
6. Verify: Allocation appears in table
7. Change OPEX to create mismatch between total and allocated
8. Verify: Warning banner shows "Allocated amount does not match declared budget"

**Test 4: Actuals Import (ACTL-01, ACTL-02, ACTL-03)**
1. Click "Upload Actuals" in portfolio top bar
2. Create test receipts Excel with: ProjectId (PRJ-XXXX format), Amount, Currency, Date
3. Upload receipts file
4. Verify: Imported count shown
5. Test with invalid ProjectId - verify error returned
6. Create test invoices Excel with: ProjectId, InvoiceNumber, Amount, Currency, Date, Description
7. Upload invoices file
8. Verify: Extraction warnings shown if patterns don't match

**Test 5: Actuals Summary (ACTL-08)**
1. Open project that has actuals imported
2. Look at actuals summary section in sidebar
3. Verify: Receipts total, invoices total, budget remaining shown
4. Verify: Progress bar reflects percent used
5. If any invoices need attention, verify alert shows count

**Test 6: Competence Month Override (ACTL-04, ACTL-05, ACTL-06)**
1. Navigate to invoices that have extraction warnings
2. Verify: Invoices show as needing attention
3. (If UI exists) Set manual competence month override
4. Verify: Invoice no longer flagged as needing attention

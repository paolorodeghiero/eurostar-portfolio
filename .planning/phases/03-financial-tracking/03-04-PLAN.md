  <name>Task 1: Create competence month extraction utility</name>
  <files>backend/src/lib/competence-month.ts</files>
  <action>
Create backend/src/lib/competence-month.ts:

Export extractCompetenceMonth function:
- Takes: db instance, description (string), company (string)
- Returns: { month: string | null, extracted: boolean }

Implementation:
1. Query competence_month_patterns table for patterns matching company
2. Try each pattern's regex against description
3. If match found with named group 'month', return { month: matchedValue, extracted: true }
4. If no match, return { month: null, extracted: false }

Example patterns table data:
- company: 'THIF', pattern: '(?<month>\\d{4}-\\d{2})', description: 'YYYY-MM format'
- company: 'EIL', pattern: 'Period:\\s*(?<month>\\d{4}/\\d{2})', description: 'Period: YYYY/MM'

The function should:
- Handle regex errors gracefully (invalid patterns should skip, not crash)
- Log pattern match attempts for debugging
- Normalize extracted month to YYYY-MM format if needed

Export normalizeMonthFormat helper:
- Converts various formats (2024/01, 01/2024, Jan 2024) to YYYY-MM
- Returns null if format unrecognized
  </action>

  <action>
Create backend/src/routes/actuals/ directory and files:

**receipts.ts:**
1. GET /api/actuals/receipts
   - List all receipts with project details (projectId as PRJ-YYYY-XXXXX format)
   - Support filters: projectId, fromDate, toDate, currency

2. POST /api/actuals/receipts/import
   - Accept JSON array of receipts: [{ projectId, receiptNumber?, amount, currency, receiptDate, description? }]
   - Validate each receipt:
     - projectId exists in projects table (lookup by PRJ-XXXX format)
     - currency exists in currency_rates table
     - amount is positive
   - Insert valid receipts, collect errors for invalid
   - Generate importBatch UUID for this import
   - Return { imported: number, errors: [{ index, message }] }

3. DELETE /api/actuals/receipts/:id
   - Delete single receipt

**invoices.ts:**
1. GET /api/actuals/invoices
   - List all invoices with project details
   - Include competenceMonthExtracted flag for alerting (ACTL-05)
   - Support filters: projectId, fromDate, toDate, extractionFailed (boolean)

2. POST /api/actuals/invoices/import
   - Accept JSON array: [{ projectId, invoiceNumber, amount, currency, invoiceDate, description }]
   - For each invoice:
     - Validate projectId exists
     - Validate currency exists
     - Extract competence month using extractCompetenceMonth
     - Set competenceMonthExtracted based on extraction success
   - Insert with importBatch UUID
   - Return { imported, errors, extractionWarnings: number }

3. PUT /api/actuals/invoices/:id/competence-month
   - Accept: { competenceMonth: 'YYYY-MM' }
   - Update competenceMonthOverride field (ACTL-06)
   - This override takes precedence in all queries

4. DELETE /api/actuals/invoices/:id
   - Delete single invoice

**index.ts:**
- Export combined actuals routes

**backend/src/index.ts:**
- Register actuals routes under /api/actuals prefix

Note: Store all amounts in original currency (ACTL-07). Currency conversion happens only in reporting views (Phase 6).

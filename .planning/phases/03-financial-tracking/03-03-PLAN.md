  <name>Task 1: Create cost T-shirt derivation utility</name>
  <files>backend/src/lib/cost-tshirt.ts</files>
  <action>
Create backend/src/lib/cost-tshirt.ts:

Export deriveCostTshirt function:
- Takes: db instance, totalBudget (string), currency (string)
- Queries cost_tshirt_thresholds table for matching currency, ordered by maxAmount ascending
- Finds first threshold where totalBudget <= maxAmount
- Returns the size (XS, S, M, L, XL, XXL)
- If no match (budget exceeds all thresholds), returns last size (XXL)
- If no thresholds exist for currency, returns null

Use PostgreSQL for comparison (not JavaScript Number):
```typescript
// Query thresholds and compare using SQL
const thresholds = await db
  .select()
  .from(costTshirtThresholds)
  .where(eq(costTshirtThresholds.currency, currency))
  .orderBy(asc(costTshirtThresholds.maxAmount));
```

Compare using parseFloat only after retrieving - or better, do comparison in SQL with CAST.

Create backend/src/routes/projects/project-budget.ts:

1. GET /api/projects/:projectId/budget
   - Return project budget fields: opexBudget, capexBudget, budgetCurrency, costTshirt
   - Include allocations with budget line details (lineValue, company, available amount)
   - Calculate and return:
     - totalBudget = opexBudget + capexBudget (as string)
     - totalAllocated = sum of allocation amounts
     - allocationMatch = (totalAllocated === totalBudget) - for BUDG-04 alert

2. PUT /api/projects/:projectId/budget
   - Accept: { opexBudget, capexBudget, budgetCurrency }
   - All amounts as strings (never convert to Number for arithmetic)
   - Calculate totalBudget in PostgreSQL: UPDATE ... SET cost_tshirt = (derive)
   - Call deriveCostTshirt and update costTshirt field
   - Increment project version for optimistic locking
   - Return updated budget with new costTshirt

3. GET /api/projects/:projectId/budget/allocations
   - List all allocations for project
   - Join budget_lines for details

4. POST /api/projects/:projectId/budget/allocations
   - Accept: { budgetLineId, allocationAmount }
   - Validate in transaction with SERIALIZABLE isolation:
     - Get budget line with SELECT ... FOR UPDATE to lock row
     - Calculate current total allocated for this budget line
     - Check if new allocation would exceed available
     - If exceeds: return 400 with { error: 'Exceeds available', available, requested }
   - Insert allocation
   - Return allocation with budget line details

5. PUT /api/projects/:projectId/budget/allocations/:budgetLineId
   - Update allocation amount with same validation as POST
   - Use SELECT FOR UPDATE on budget line

6. DELETE /api/projects/:projectId/budget/allocations/:budgetLineId
   - Remove allocation (no validation needed - always allowed)

Register in projects/index.ts under the projects prefix.

Key: Use db.transaction for allocation changes to prevent race conditions (BUDG-03).

cd backend && npm install https://cdn.sheetjs.com/xlsx-0.20.3/xlsx-0.20.3.tgz @fastify/multipart zod

Create backend/src/lib/excel-parser.ts with:

1. Import xlsx library (import * as XLSX from 'xlsx')
2. Import zod for validation

3. Export budgetLineRowSchema using Zod:
```typescript
const budgetLineRowSchema = z.object({
  Company: z.string().min(1),
  Department: z.string().min(1),
  CostCenter: z.string().min(1),
  LineValue: z.string().min(1),
  Amount: z.number().positive(),
  Currency: z.string().length(3),
  Type: z.enum(['CAPEX', 'OPEX']),
  FiscalYear: z.number().int().min(2020).max(2100),
});
```

4. Export parseExcelBuffer function:
- Takes Buffer, returns array of parsed rows
- Uses XLSX.read(buffer, { type: 'buffer' })
- Gets first sheet, converts to JSON with XLSX.utils.sheet_to_json
- Returns raw data array

5. Export validateExcelFile function:
- Takes Buffer, checks magic bytes (504b0304 for xlsx, d0cf11e0 for xls)
- Returns { valid: boolean, error?: string }

6. Export validateBudgetLineRows function:
- Takes raw data array
- Maps each row through budgetLineRowSchema.safeParse
- Returns { valid: Row[], errors: Array<{ row: number, message: string }> }

  <files>

  <action>
Create backend/src/routes/admin/budget-lines.ts following the departments.ts pattern:

1. GET / - List all budget lines with joins:
   - Join departments to show department name
   - Join cost_centers to show cost center code
   - Include allocated amount (sum from project_budget_allocations)
   - Include available amount (lineAmount - allocated)
   - Support query params: fiscalYear, company, type

2. GET /:id - Get single budget line with:
   - Department and cost center details
   - List of projects using this budget line (from allocations)

3. POST /import - Excel import endpoint:
   - Register @fastify/multipart plugin in this file
   - Accept multipart file upload
   - Validate file with validateExcelFile (magic bytes)
   - Parse with parseExcelBuffer
   - Validate rows with validateBudgetLineRows
   - For each valid row:
     - Look up departmentId by name (error if not found)
     - Look up costCenterId by code (error if not found)
     - Validate currency exists in currency_rates (error if not found)
   - Insert all valid rows in transaction
   - Return { imported: number, errors: Array<{ row, message }> }

4. DELETE /:id - Delete budget line:
   - Check if any allocations exist (count from project_budget_allocations)
   - If allocations > 0, return 409 with usage count
   - Otherwise delete

Register in referentials.ts under /api/admin/budget-lines prefix.

Note: No PUT/update - budget lines are imported, not manually edited. To fix errors, delete and re-import.
  </action>

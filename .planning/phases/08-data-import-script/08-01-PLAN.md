<task type="auto">
  <name>Task 2: Add import tracking columns to projects table and create date parser</name>
  <files>
    backend/src/db/schema.ts
    backend/import/scripts/lib/date-parser.ts
  </files>
  <action>
    1. Add import tracking columns to projects table in schema.ts:
       ```typescript
       // After createdBy/updatedBy columns, add:
       importedAt: timestamp('imported_at'),  // When project was imported
       importSource: varchar('import_source', { length: 100 }),  // Source file name
       ```

       These columns are nullable to distinguish imported projects from manually created ones.

    2. Create `backend/import/scripts/lib/date-parser.ts`:

       Implement `parseFlexibleDate(value: string | number): string | null` that handles:
       - Excel serial date numbers (e.g., 44927 -> "2023-01-01")
       - Quarter format: "Q1 2026" or "Q1-2026" -> "2026-01-01" (start of quarter)
       - Year only: "2026" -> "2026-01-01"
       - Year-month: "2026-06" -> "2026-06-01"
       - ISO date: "2026-06-15" -> "2026-06-15" (passthrough)
       - Empty/null: return null

       Include mapping for quarter to month:
       - Q1 -> 01, Q2 -> 04, Q3 -> 07, Q4 -> 10

       Export the function as named export.

    3. After schema change, run migration:
       ```bash
       cd backend && npm run db:push
       ```
  </action>
  <verify>
    - `grep "importedAt" backend/src/db/schema.ts` shows column definition
    - `grep "importSource" backend/src/db/schema.ts` shows column definition
    - `npx tsx backend/import/scripts/lib/date-parser.ts` compiles without error
    - Database has imported_at and import_source columns (check via db:push output)
  </verify>
  <done>
    Projects table has importedAt and importSource columns. Date parser handles Q1-Q4, year, year-month, ISO, and Excel serial formats.
  </done>
</task>

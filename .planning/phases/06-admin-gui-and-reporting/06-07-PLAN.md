1. Swagger/OpenAPI documentation with Eurostar branding

3. Admin enhancements: usage visibility, audit log, bulk import/export
4. API link in navbar, UsageDrawer, AlertDialog confirmations
  </what-built>
  <how-to-verify>
**Criterion 1: Admin GUI displays list of all referential types with CRUD tables per type**
- Navigate to /admin
- Verify sidebar shows all 9+ referential types
- Click each type, verify table with data and CRUD actions

**Criterion 2: Admin can see which projects use each referential item**
- Go to /admin/departments
- Click "View Usage" on any department
- Verify drawer shows projects using that department
- Repeat for Teams, Statuses

**Criterion 3: PostgreSQL views in reporting schema**
- Connect to database via psql or pgAdmin
- Run: `SELECT table_name FROM information_schema.views WHERE table_schema = 'reporting';`
- Verify 10+ dim_* and 3 fact_* views exist

**Criterion 4: Currency conversion in views**
- Run: `SELECT project_id, amount_original, original_currency, amount_eur FROM reporting.fact_receipts LIMIT 10;`
- Verify amount_eur is populated for receipts with matching currency rates

**Criterion 5: Power BI DirectQuery**
- Open Power BI Desktop
- Connect to PostgreSQL with reporting schema
- Verify views are accessible and queryable

**Criterion 6: OpenAPI/Swagger documentation**
- Navigate to /docs
- Verify Swagger UI loads with Eurostar branding (teal header, logo)
- Expand any endpoint, verify description and schemas
- Click "Try It Out", verify it works with authentication

**Criterion 7: API endpoints require EntraID authentication**
- Open browser DevTools Network tab
- Access /docs without session - verify 401
- Access /api/admin/* without token - verify 401
- With valid token, verify 200 responses
  </how-to-verify>

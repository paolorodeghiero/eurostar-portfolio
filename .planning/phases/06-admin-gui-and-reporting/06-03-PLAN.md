  <action>
Add GET /:id/usage endpoint to each referential route file:

1. departments.ts - GET /:id/usage:
   - Find projects where lead_team belongs to this department
   - Query: projects JOIN teams WHERE teams.department_id = :id
   - Return: { projects: [{ id, projectId, name, statusName }] }

2. teams.ts - GET /:id/usage:
   - Find projects where team is lead or involved
   - Query: projects WHERE lead_team_id = :id UNION project_teams WHERE team_id = :id
   - Return: { projects: [{ id, projectId, name, statusName, role: 'lead' | 'involved' }] }

3. statuses.ts - GET /:id/usage:
   - Find projects with this status
   - Query: projects WHERE status_id = :id
   - Return: { projects: [{ id, projectId, name }] }

4. outcomes.ts - GET /:id/usage:
   - Find projects with scores for this outcome
   - Query: projects JOIN project_values WHERE outcome_id = :id
   - Return: { projects: [{ id, projectId, name, score }] }

5. cost-centers.ts - GET /:id/usage:
   - Find budget lines using this cost center
   - Query: budget_lines WHERE cost_center_id = :id
   - Also find projects via allocations
   - Return: { budgetLines: [...], projects: [...] }

Each endpoint should:
- Return 404 if referential item not found
- Return empty arrays if no usage
- Include project statusName for context
  </action>

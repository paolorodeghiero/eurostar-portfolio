  <action>
Install @fastify/swagger@^9.7.0, @fastify/swagger-ui@^5.0.0, and fastify-type-provider-zod@^5.0.0.

Create backend/src/plugins/swagger.ts with:
- OpenAPI 3.0.3 spec configuration
- API info: title "Eurostar Portfolio API", version "1.0.0", description
- Server URL: /api (existing API prefix)
- Security scheme: BearerAuth (http, bearer, JWT format, "EntraID JWT token" description)
- Tags for resource grouping: Projects, Departments, Teams, Statuses, Outcomes, Cost Centers, Currency Rates, Thresholds, Budget Lines, Actuals, Alerts, Admin
- Configure response schemas to document error scenarios per endpoint:
  - 400: Validation errors (Zod schema violations)
  - 401: Unauthorized (missing/invalid token)
  - 403: Forbidden (insufficient permissions)
  - 404: Resource not found
  - 500: Internal server error
  The Zod schemas already define these; ensure they appear in OpenAPI output with clear descriptions.
- Register @fastify/swagger with jsonSchemaTransform from fastify-type-provider-zod
- Register @fastify/swagger-ui at /docs with:
  - Custom CSS for Eurostar theme:
    - .swagger-ui .topbar { background-color: #086264; }
    - .swagger-ui .opblock-tag { color: #086264; }
    - .swagger-ui .btn.authorize { background-color: #086264; border-color: #086264; }
    - .swagger-ui .opblock .opblock-summary { border-left: 3px solid #086264; }
    - Body font: system-ui, -apple-system, sans-serif
  - Logo configuration using eurostar-logo.png
  - uiConfig: docExpansion 'list', deepLinking true, displayRequestDuration true, filter true, tryItOutEnabled true
  - staticCSP: false (to allow custom CSS)

Copy eurostar-logo.png from project root to backend/assets/ directory. Create assets/ directory if needed.

Export as fastify-plugin wrapped function.

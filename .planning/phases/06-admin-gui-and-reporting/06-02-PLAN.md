  <action>
Create backend/src/db/reporting-views.ts with a createReportingViews function that:

1. Creates 'reporting' schema if not exists

2. Creates 9 dimension views (all referential types):
   - dim_departments: department_key, department_name, created_at, updated_at
   - dim_teams: team_key, team_name, department_key, department_name, created_at, updated_at
   - dim_statuses: status_key, status_name, color, display_order, is_system_status, is_read_only
   - dim_outcomes: outcome_key, outcome_name, score_examples (JSON or separate columns)
   - dim_cost_centers: cost_center_key, code, description
   - dim_currency_rates: rate_key, from_currency, to_currency, rate, valid_from, valid_to
   - dim_committee_thresholds: threshold_key, min_amount, max_amount, level, currency
   - dim_cost_tshirt_thresholds: threshold_key, size, max_amount, currency
   - dim_competence_month_patterns: pattern_key, company, pattern, description

3. Create time dimension view:
   - dim_date: date_key, year, quarter, month, day, month_name, day_name
   - Use generate_series from 2020-01-01 to 2030-12-31
   - Fiscal year = calendar year per user decision

4. Creates 3 fact views:
   - fact_projects (summary):
     - project_key, project_id, project_name
     - status_key, lead_team_key
     - start_date, end_date, start_year, start_quarter, start_month
     - opex_budget_eur, capex_budget_eur, total_budget_eur (already in EUR in database)
     - actuals_total_eur (SUM from receipts, converted)
     - budget_remaining_eur (total_budget - actuals_total)
     - percent_used ((actuals_total / total_budget) * 100)
     - days_until_end (end_date - CURRENT_DATE, NULL if no end_date or past)
     - committee_state, committee_level, cost_tshirt
     - is_read_only (from status join)
     - created_at, updated_at

   - fact_receipts:
     - receipt_key, project_key, project_id
     - receipt_number, company, purchase_order
     - amount_original, original_currency
     - amount_eur (converted using currency_rates with valid date range)
     - receipt_date, receipt_year, receipt_quarter, receipt_month
     - created_at

   - fact_invoices:
     - invoice_key, project_key, project_id
     - invoice_number, company, purchase_order
     - amount_original, original_currency
     - amount_eur (converted)
     - invoice_date, invoice_year, invoice_quarter, invoice_month
     - competence_month, competence_month_extracted
     - created_at

5. Use LEFT JOIN with currency_rates for conversion:
   - Match from_currency to receipt/invoice currency
   - Match to_currency = 'EUR'
   - Check valid_from <= date AND (valid_to IS NULL OR valid_to >= date)
   - If no rate found, keep NULL for amount_eur

6. Use CREATE OR REPLACE VIEW for idempotency

Export function createReportingViews(db) that executes all view creation SQL.

Create backend/vitest.config.ts:
- Use defineConfig from 'vitest/config'
- Set environment: 'node' (backend runs in Node, not browser)
- Configure coverage with v8 provider, reporters: ['text', 'json', 'html']
- Include src/**/*.ts in coverage, exclude test files, __tests__, types, migrations
- Set setupFiles pointing to './src/__tests__/setup.ts'
- Enable globals: true for describe/test/expect without imports
- Set testTimeout: 10000 for database operations

Create backend/test.env:
```
TEST_DATABASE_URL=postgres://localhost:5432/eurostar_portfolio_test
DEV_MODE=true
```
Note: DEV_MODE=true skips auth validation during tests.

Extract Fastify app builder from server.ts into backend/src/app.ts:
- Export `build(opts?: FastifyServerOptions)` function that creates and configures Fastify instance
- Move all plugin registrations (cors, swagger, auth, db, routes) into build function
- Accept optional databaseUrl parameter for test database connection
- Keep server.ts slim: just call build() and listen()
- This separation is critical for Fastify HTTP injection testing

Update backend/src/server.ts:
- Import build from './app'
- Call build() to get app instance
- Call app.listen() as before

Create backend/src/__tests__/setup.ts:
- Import dotenv and load test.env: `dotenv.config({ path: './test.env' })`
- Add beforeAll hook to verify TEST_DATABASE_URL is set
- Add placeholder comments for database reset/seed utilities (to be added in Plan 02)
- Export getTestApp helper that calls build({ logger: false, databaseUrl: process.env.TEST_DATABASE_URL })
